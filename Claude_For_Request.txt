请用一个单独的 HTML 文件（可内嵌 CSS 与 JS，可使用 CDN 库）实现一个基于 WebGL / Canvas 的网页版"船舵操控模拟器"，用于帮助学习者通过实际体验理解 PID 控制原理的起源——水手手动掌舵控制航向的过程。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【一、核心目标与教学背景】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
本模拟器还原经典 PID 思想的物理原型：水手依靠观察偏航误差（Heading Error）手动转动船舵，以使船舶沿目标航向行驶。用户扮演水手，通过操作舵角体验：
  · 系统输出（航向）对输入（舵角）的滞后性与惯性
  · 误差永不为零时需持续施加控制力的本质
  · 手动调节的疲惫感，从而理解自动控制的必要性

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【二、物理仿真引擎（强制要求）】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. 航行运动学：
   · 船体具有偏航角惯性矩（yaw inertia），舵角变化不立即改变航向，存在真实响应延迟（约 1~3 秒到位）
   · 舵效与船速正相关：低速时转向力弱，高速时响应更灵敏
   · 水阻尼效应：无舵输入时偏转角速度自然衰减
   · 风力干扰：随机方向/强度的侧风持续对船施加偏转力矩，强度可调（0~强）
   · 洋流：可选择性加入恒定方向横流
   · 船速可在 3 档（慢 / 中 / 快）切换，影响舵效与场景滚动速度

2. 舵角控制：
   · 键盘 A/D 键或左右方向键：连续增减舵角，最大 ±35°
   · 鼠标拖拽屏幕中心的舵轮图标（第一视角模式）：顺/逆时针旋转赋值舵角
   · 舵角变化速率有物理限制（不能瞬间到达极值），模拟真实液压舵机
   · 按 S 键或空格：舵角快速归零（回舵）

3. 数值模型（参考，可做近似）：
   · 偏航角速度 dψ/dt = K_r · δ(t) · v - D · ψ̇（K_r 为舵效系数，δ 为舵角，v 为船速，D 为阻尼）
   · 风扰力矩 τ_wind = A_wind · sin(ω_wind · t + φ)（随机参数）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【三、3D 场景渲染（强制要求）】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
使用 Three.js（从 CDN 加载）渲染以下场景：

1. 海面：
   · 使用顶点动画或着色器实现动态波浪，波高与风速关联
   · 海面颜色随时段变化（见下）
   · 海面有高光反射效果（Phong/PBR 近似）
   · 随机分布少量白色浪花泡沫粒子

2. 天空与时段（用户可在界面切换，强制要求三种）：
   · 晨曦（Morning）：天空橙粉渐变，太阳低角，水面橘红倒影，薄雾感
   · 正午（Noon）：天空深蓝，太阳高位白光，水面深蓝绿色，高对比度
   · 夕阳（Sunset）：天空金橙紫渐变，太阳接近地平线，水面火焰倒影，最具美感

3. 船只模型：
   · 第三人称：渲染一艘简洁风格的帆船或货轮（可用基本几何体组合，无需复杂模型）
   · 船体随波浪有轻微横摇/纵摇动画（±2~5°）
   · 船尾有尾迹（白色泡沫带，随速度变长）

4. 场景元素（增强沉浸感）：
   · 远处随机分布 2~4 艘静止小船/岛屿轮廓，作为视觉参照
   · 天空中慢速飘动的云朵（3~6 个简单几何体云）
   · 若为夕阳/晨曦模式，天空添加太阳光晕（Lens Flare 效果或简单发光圆）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【四、视角系统（强制要求）】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
按 V 键或界面按钮切换，平滑过渡（0.5s tween）：

1. 第三人称（默认）：
   · 摄像机跟随船只，位于船只后上方（约 30~45° 俯角，距船 20~30m）
   · 摄像机始终对准船头方向
   · 可用鼠标右键拖拽绕船旋转视角

2. 第一人称（船长视角）：
   · 摄像机置于船桥/甲板前方，视线与海平面近似水平
   · 屏幕中央显示可交互的舵轮（SVG 或 3D 模型），鼠标拖拽旋转控舵
   · 展示船头摇摆的视觉效果（随偏转角轻微晃动摄像机）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【五、目标航向系统与导航辅助】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. 用户可通过界面（滑块或输入框）设定目标航向（0°~360°，北为 0°）
2. 场景中显示一条半透明的目标航线（延伸至远方的绿色虚线）
3. 屏幕顶部或左上角显示：
   · 罗盘（Compass）：实时旋转，标注当前航向与目标航向
   · 偏差角度（Heading Error）：醒目红色数字显示，正负号表示左右偏
   · 当前舵角指示表盘（-35° ~ +35°）
   · 当前船速（节，knots）
4. 可选：地图小窗（minimap）右上角，显示船迹轨线与目标方向

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【六、底部状态与 PID 信息面板（强制要求）】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
页面底部固定一个信息面板（高度约 200px，不遮挡主视口，半透明深色背景），分为以下区域：

1. 实时折线图（主图，占面板约 60% 宽）：
   · X 轴：时间（滚动，显示最近 60 秒）
   · 实时绘制三条曲线：
     - 目标航向（绿色虚线，水平参考线）
     - 当前航向输出（蓝色实线）
     - 偏航误差 e(t)（红色实线，相对于零轴）
   · 曲线区域标注"系统响应"字样
   · 纵轴单位为度（°）

2. 数值状态栏（面板右侧，约 40% 宽，网格排列）：
   · 目标航向 ψ_ref（°）
   · 当前航向 ψ（°）
   · 偏差 e = ψ_ref - ψ（°）
   · 当前舵角 δ（°）
   · 积分累计误差 ∫e dt（°·s，实时计算，用于理解积分含义）
   · 误差变化率 de/dt（°/s，用于理解微分含义）
   · 操控评分（0~100，根据偏差大小和持续时间扣分）

3. 教学提示区（面板底部，小字）：
   · 根据当前误差状态动态生成提示文字，如：
     "当前存在持续偏差，单纯靠比例控制难以消除——这正是积分控制存在的原因"
     "误差快速增大，此时微分控制将预测趋势并提前抑制"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【七、操作说明与 UI 控件】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
界面左侧或顶部工具栏（可折叠）：
  · 时段选择：Morning / Noon / Sunset 三个按钮

  · 视角切换按钮（同 V 键）
  · 操控说明悬浮卡：键位图示（A/D 转舵，S 回舵，V 切视角）

【八、技术规范与实现要求】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. 单一 HTML 文件，内嵌所有 CSS 和 JS，仅允许通过 CDN 加载：
   · Three.js（r128 或更新稳定版）
   · 可选：Tween.js（视角切换平滑过渡）

4. 无任何外部图片依赖，所有视觉元素用 Three.js 几何体 + 材质实现
5. 代码结构清晰，分模块注释（Physics / Renderer / UI / Controls）
