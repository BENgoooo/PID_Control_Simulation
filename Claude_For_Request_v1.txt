【新需求】

目前已有版本虽然能够让学习者体验驾驶轮船和手动掌舵，但 PID 控制器在整个航向控制过程中的作用仍然不够直观，学习者难以仅凭航行体验理解比例、积分和微分三部分分别解决了什么问题。

请在原有船舵操控模拟器的基础上增加两种明确分离的运行模式：

1. 体验模式
   · 以沉浸式手动驾驶为核心，继续由用户使用键盘或鼠标直接操纵船舵。
   · 全面优化画面、控制手感和操作反馈，将整体视觉质量提升至超高水平。
   · 强化海浪、天空、光照、水面反射、浪花、船体、尾迹、天气、镜头和环境音效等表现。
   · 优化舵轮阻尼、液压舵机延迟、船体惯性、震动、镜头反馈和视觉反馈，使操控既真实又易于感知。
   · 在保持物理真实性的同时，使用户能够清楚感受到转舵、船体响应、惯性延续、风流扰动和回舵之间的关系。

2. 教学模式
   · 以循序渐进理解 PID 为核心，不应只是展示 PID 数值，而要让学习者实际观察、对比和操作。
   · 按照“手动控制基线 → 比例控制 P → 比例积分控制 PI → 比例积分微分控制 PID”的顺序逐步解锁。
   · 每个阶段使用相同或可复现的目标航向、风力、洋流和初始条件，使不同控制器之间可以公平对比。
   · P 阶段需要直观展示：误差越大，比例输出越大；在持续外部扰动下，为了维持抵消扰动所需的非零舵角，系统通常必须保留一定稳态误差。
   · PI 阶段需要直观展示：积分项会累积持续误差，使系统即使在误差趋近于零时仍能保留必要控制量，从而消除稳态偏差；同时展示积分带来的超调、响应变慢和积分饱和风险。
   · PID 阶段需要直观展示：微分项根据误差变化趋势或航向变化率提供提前抑制，减少超调和振荡；同时展示微分对噪声敏感、需要滤波的特点。
   · 实时拆分显示 P、I、D 三项各自产生的舵角贡献，以及它们合成后的最终舵角指令。
   · 提供分步任务、动态讲解、参数调节、同场景重放、曲线叠加和阶段总结，使学习者能够通过实验得出结论。

注意教学表述必须严谨：不能笼统宣称“所有只使用比例控制的系统都永远存在误差”。应结合本船舶模型说明，在持续风力或洋流等恒定扰动存在时，P 控制器需要依靠非零误差产生持续的抵消舵量，因此会形成稳态偏差；实际稳态误差还与被控对象、系统类型和扰动形式有关。

最终作品仍需满足原版 Prompt 中关于单一 HTML 文件、真实航行物理、第一/第三视角、三种时段、目标航向、实时曲线、PID 状态信息和响应式布局的全部要求。

【融合后的总 Prompt】

请用一个单独的 HTML 文件实现一个基于 Three.js / WebGL / Canvas 的网页版“船舵操控与 PID 教学模拟器”。HTML 文件可内嵌全部 CSS 与 JavaScript，可通过 CDN 加载稳定版开源库，但不能依赖任何外部图片、模型或音频文件。

模拟器需要以“水手通过观察航向误差并操纵船舵”这一经典自动控制原型为背景，同时兼顾高质量驾驶体验和严谨的 PID 交互教学。最终作品必须是可直接运行、功能完整的成品，而不是静态界面、概念演示图或代码片段。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【一、核心目标与教学原则】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 还原船舶航向控制的关键物理特征：
   · 舵角变化不会立即改变航向。
   · 船体具有偏航惯性，回舵后仍会继续转动。
   · 舵效随船速变化。
   · 水阻尼会逐步衰减偏航角速度。
   · 风力、阵风和洋流会持续造成航向或航迹扰动。
   · 用户需要观察误差、判断趋势并提前操作，而不是简单让船头瞬间对准目标。

2. 让学习者通过可重复实验理解：
   · P 使用当前误差产生控制作用。
   · I 使用误差的历史累计补偿持续偏差。
   · D 使用误差或测量值的变化趋势抑制超调和振荡。
   · P、I、D 各自的贡献、优点、缺陷和相互配合方式。

3. 教学内容必须准确：
   · 不得笼统宣称“所有 P 控制系统都必然存在稳态误差”。
   · 在本船舶模型中，应通过恒定侧风偏航力矩或其他持续扰动说明：若系统需要一个非零舵角来抵消扰动，P 控制器必须保留非零误差才能输出该舵角，因此会形成稳态偏差。
   · PI 消除该偏差的原因应解释为：积分项记录并累积过去的误差，在当前误差接近零后仍能维持抵消扰动所需的控制量，而不是简单描述为“把缺少的数值填上”。
   · D 项应说明为对趋势的阻尼或预测性修正，并展示噪声放大和滤波的必要性。

【二、双模式总体结构，强制要求】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

首次进入页面时显示精致的模式选择界面，也可在运行中随时切换：

1. 体验模式 Experience
   · 用户完全手动操纵船舵，以沉浸式驾驶、真实物理和高质量画面为核心。
   · PID 自动控制默认关闭，但仍可显示航向误差、积分累计量和误差变化率，帮助用户建立直觉。
   · 强化操控反馈、镜头反馈、风浪反馈、舵机反馈、航迹反馈和评分系统。
   · 提供自由航行、航向保持挑战、阵风挑战、横流挑战和狭窄航道挑战等场景预设。

2. 教学模式 Tutorial
   · 采用课程式流程，按“手动基线 → P → PI → PID”逐步进行。
   · 未完成当前阶段前可以允许预览后续阶段，但界面应明确推荐学习顺序。
   · 每一阶段包含：概念说明、预测问题、开始实验、实时观察、参数调整、结果对比、结论总结。
   · 用户可暂停、单步、重置、以 0.5× / 1× / 2× 时间速度运行实验。
   · 每次实验可使用固定随机种子，确保目标航向、风力和洋流完全一致，便于公平对照。
   · 提供“重新运行上一控制器”和“叠加对比全部阶段”功能。

模式切换时保留设置，但教学实验的历史数据必须与自由体验数据分开管理。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【三、教学模式课程设计，最高优先级】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

教学模式至少包含以下四个连续阶段：

1. 阶段 0：手动控制基线
   · 用户使用 A/D、左右方向键或舵轮手动保持目标航向。
   · 先在无风条件下体验响应延迟和偏航惯性，再开启恒定侧风。
   · 记录平均绝对误差、最大误差、舵轮操作次数、过零次数和控制能量。
   · 通过提示引导用户观察：“误差出现后才修正”“回舵过晚导致超调”“持续扰动需要持续操作”。
   · 完成后询问：能否让控制器根据误差自动转舵？

2. 阶段 1：比例控制器 P
   · 自动舵角指令公式：
     u_P(t) = Kp · e(t)
     δ_cmd = clamp(u_P, -35°, +35°)
   · 提供 Kp 滑块、推荐值、过小/适中/过大三个快速预设。
   · 场景先运行无扰动阶跃航向实验，再加入恒定侧风偏航力矩。
   · 实时用颜色和箭头表示误差 e 如何直接生成 P 输出。
   · Kp 过小时展示响应慢、偏差大；Kp 过大时展示超调、摆动或频繁修正。
   · 在持续扰动下稳定后，冻结画面并突出显示：
     - 当前仍存在的稳态误差 e_ss
     - P 输出 Kp·e_ss
     - 抵消扰动所需要的非零舵角
   · 动态说明：“如果误差完全为零，P 输出也会变为零，便无法继续抵消当前恒定扰动，因此系统停在一个非零误差处。”

3. 阶段 2：比例积分控制器 PI
   · 自动舵角指令公式：
     u_P(t) = Kp · e(t)
     u_I(t) = Ki · ∫e(t)dt
     δ_cmd = clamp(u_P + u_I, -35°, +35°)
   · 提供 Kp、Ki 参数滑块和推荐预设。
   · 使用与 P 阶段完全相同的扰动与初始条件。
   · 实时可视化积分容器或“误差蓄水槽”：正误差逐步累积，负误差逐步消减。
   · 明确展示当 e 接近 0 时，P 输出接近 0，但 I 输出仍能保留抵消风力所需的舵角。
   · 展示 Ki 过小时消除静差很慢，Ki 过大时容易超调、振荡和积分饱和。
   · 必须实现舵角饱和和 anti-windup，可在界面切换“抗积分饱和开/关”进行对比。
   · anti-windup 可采用积分钳位或反算方法，并在代码注释和教学说明中写清楚。

4. 阶段 3：完整 PID
   · 自动舵角指令公式：
     u_P(t) = Kp · e(t)
     u_I(t) = Ki · ∫e(t)dt
     u_D(t) = Kd · de(t)/dt
     δ_cmd = clamp(u_P + u_I + u_D, -35°, +35°)
   · 提供 Kp、Ki、Kd 参数滑块，以及保守、平衡、激进三个预设。
   · D 项应使用低通滤波后的误差变化率，或使用测量值微分以降低目标值阶跃导致的 derivative kick。
   · 提供“微分滤波强度”参数，并可切换显示原始微分值和滤波后微分值。
   · 使用与 P、PI 相同的实验，突出展示 D 项在接近目标、误差快速变化时产生反向抑制，减少超调和稳定时间。
   · Kd 过小时阻尼不足；Kd 过大时展示噪声敏感、舵机抖动和控制输出不平滑。
   · 最终用同一张曲线叠加 P、PI、PID 的航向响应，并比较：
     - 上升时间 Rise Time
     - 最大超调量 Overshoot
     - 调节时间 Settling Time
     - 稳态误差 Steady-State Error
     - IAE：∫|e|dt
     - 控制能量：∫u²dt

5. 教学交互要求
   · 每个阶段开始前提出一个简短预测问题，例如“增大 Kp 会让船更快稳定还是更容易摆动？”
   · 实验进行时根据状态给出动态提示，但不要直接替用户完成参数选择。
   · 关键现象发生时允许自动慢放或暂停，并在场景和曲线上同时标注原因。
   · 每阶段结束后生成一张结论卡，包含观察结果、正确解释、常见误区和进入下一阶段按钮。
   · 提供“自由调参实验台”，允许任意启用/禁用 P、I、D 三项。
   · 提供“隐藏公式”选项，便于先凭感觉实验，再显示数学解释。

【四、控制器可视化与对比系统，强制要求】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 增加控制器分量面板：
   · P 输出 uP，使用橙色。
   · I 输出 uI，使用紫色。
   · D 输出 uD，使用青色。
   · 合成输出 u = uP + uI + uD，使用白色。
   · 饱和后的实际舵角指令 δ_cmd，使用黄色。
   · 液压舵机实际到达的舵角 δ_actual，使用蓝色。

2. 使用横向矢量条或力的合成图表现各控制量：
   · 正值指向右舷，负值指向左舷。
   · 能清楚看到 P、I、D 是相互叠加还是相互抵消。
   · 舵角饱和时明确显示“SATURATION”。

3. 实时曲线至少支持以下开关：
   · 目标航向 ψ_ref
   · 当前航向 ψ
   · 航向误差 e
   · P 输出
   · I 输出
   · D 输出
   · 指令舵角
   · 实际舵角
   · 偏航角速度

4. 对比模式：
   · 可保存当前实验并用同一扰动种子重新运行另一个控制器。
   · 能将 P、PI、PID 的航向曲线以不同颜色叠加。
   · 可在关键位置显示超调峰值、进入稳定带时刻和最终稳态误差。
   · 提供并排数据表，自动总结哪个控制器响应更快、哪个静差更小、哪个舵机动作更剧烈。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【五、物理仿真引擎，强制要求】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 使用固定物理步长 Δt = 1/60 s，并通过 accumulator 与 requestAnimationFrame 解耦。

2. 船舶航向动力学至少包含：
   · 船体偏航惯性矩或等价的一阶/二阶 Nomoto 模型。
   · 舵效与船速正相关。
   · 水动力阻尼。
   · 舵机速度限制和最大舵角 ±35°。
   · 舵轮指令角、指令舵角和实际舵角之间的响应延迟。
   · 回舵后船体因偏航惯性继续转动。

3. 可参考以下近似模型，也可使用更稳定且更真实的 Nomoto 模型：
   · dψ/dt = r
   · T·dr/dt + r = K·δ·v + τ_wind + τ_current
   · ψ 为航向角，r 为偏航角速度，δ 为实际舵角。

4. 风力干扰：
   · 包含缓慢变化风、随机阵风和可选恒定偏航力矩。
   · 风力等级 0~3 可调。
   · 教学模式支持固定随机种子，体验模式可使用随机种子。
   · 风向和风力应有场景反馈，如风向带、海面波纹、旗帜或粒子。

5. 洋流：
   · 不仅影响航向，也应使船舶实际航迹产生横向漂移。
   · 在小地图上区分“船头指向”和“实际速度方向”，用于解释 heading 与 course over ground 的差别。

6. 船速三档：
   · 慢速、中速、快速。
   · 同时影响舵效、前进速度、尾迹长度、浪花数量和镜头运动感。

7. 数值安全：
   · 对舵角、积分项和控制器输出进行合理限制。
   · 对 D 项进行滤波。
   · 切换控制器、重置实验或大幅修改目标航向时，正确处理积分状态和微分突变。
   · 航向误差必须采用最短角差，正确处理 359° 与 1° 的跨零情况。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【六、体验模式的超高画质要求，强制要求】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

体验模式的画面不能停留在简单基础几何体演示水平，应尽可能达到高质量网页 3D 小游戏的视觉观感，同时保持可运行性能。

1. 海面
   · 使用多频率 Gerstner Wave、分层正弦波或自定义顶点着色器生成大浪和细浪。
   · 实现 Fresnel 反射、天空颜色反射、太阳高光带、深浅水色变化和距离雾化。
   · 波高、波长、波速和泡沫强度随风力变化。
   · 船头破浪处生成动态浪花，船尾生成双股尾迹和逐渐消散的泡沫。
   · 海面远处与天空地平线自然融合，避免明显平面边缘。

2. 天空和大气
   · 晨曦：橙粉渐变、低位太阳、薄雾、暖色水面反光。
   · 正午：深蓝天空、高位太阳、清晰高对比度、水面蓝绿色。
   · 夕阳：金橙紫渐变、接近地平线的太阳、长条火焰倒影和最强氛围表现。
   · 程序化生成 3D 云层或多层云团，具有缓慢运动、明暗层次和远近视差。
   · 晨曦和夕阳加入太阳光晕、泛光近似与空气透视。
   · 可加入轻微色调映射、暗角和可关闭的后期处理，但不得牺牲 UI 清晰度。

3. 船只
   · 使用程序化几何体构建细节更丰富的船体、甲板、船桥、窗户、栏杆、雷达、桅杆、灯具和可动船舵。
   · 材质应区分金属、玻璃、木材、喷漆船体和湿润表面。
   · 船体随波浪产生合理横摇、纵摇和升沉，不应只是固定正弦摇摆。
   · 船头转向、浪击和高速航行时有适量水花反馈。

4. 环境
   · 远处有岛屿轮廓、灯塔、浮标、其他船只和飞行海鸟剪影等程序化参照物。
   · 浮标可作为航线挑战目标，并随波浪运动。
   · 场景元素应有层次和距离雾，避免均匀随机散布。

5. 镜头与反馈
   · 第一/第三视角使用约 0.5~0.8 秒平滑过渡。
   · 第三视角可用鼠标右键绕船旋转，滚轮调整有限范围距离。
   · 第一视角包含船桥或甲板前景、可拖拽舵轮、仪表和船头。
   · 镜头根据速度、浪击、横摇和转向产生克制的运动反馈，并提供“减少镜头晃动”选项。
   · 大舵角、舵角到限位、回舵和阵风出现时提供清晰但不过度的视觉反馈。

6. 音效
   · 不允许依赖外部音频文件。
   · 可使用 Web Audio API 程序化生成海浪、风声、低频引擎声、舵机声和提示音。
   · 必须提供静音按钮和音量调节；首次用户交互后才启动音频。

7. 性能
   · 提供低/中/高/超高四档画质，体验模式默认高或自动。
   · 根据设备像素比和帧率自适应调整粒子数量、阴影、海面网格密度和后期效果。
   · UI 中显示可选 FPS，连续低帧率时给出降档建议。
   · 目标为桌面端 60 FPS，并保证中低性能设备仍可操作。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【七、舵角控制与操控手感，强制要求】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 键盘：
   · A/D 或左右方向键连续改变舵轮/舵角指令。
   · 最大舵角 ±35°。
   · S 或空格快速回舵至中位。
   · V 切换第一/第三视角。
   · P 暂停教学实验，R 重置当前实验。

2. 鼠标和触摸：
   · 第一视角可直接拖拽舵轮顺时针或逆时针旋转。
   · 舵轮允许多圈或有限圈数旋转，并映射至 ±35° 舵角。
   · 拖拽过程应平滑，具有轻微阻尼和限位反馈。
   · 支持触屏拖拽。

3. 手感反馈：
   · 区分用户舵轮输入、控制器输出、液压舵机指令和实际舵角。
   · 舵机不能瞬间到位，满舵约 1~3 秒到位。
   · 回舵速度可以略快于普通转舵，但不能瞬移。
   · 舵角到达限位时显示限位标记并产生轻微提示反馈。
   · 教学模式自动控制时，舵轮应同步转动，直观显示“控制器正在替水手掌舵”。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【八、3D 场景、视角与导航系统】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 第三人称默认视角：
   · 摄像机位于船只后上方，约 30~45° 俯角，距离约 20~30 m。
   · 摄像机随船平滑移动并对准船体。
   · 支持有限范围轨道观察，不得穿入船体或海面。

2. 第一人称船长视角：
   · 摄像机位于船桥或甲板前部，视线接近海平面。
   · 屏幕下方显示可交互舵轮和必要仪表。
   · 能感受到船头随偏航、横摇和纵摇运动。

3. 导航辅助：
   · 用户可通过输入框和滑块设置 0°~359° 目标航向，正北为 0°。
   · 场景显示半透明目标航线或航向门。
   · 罗盘实时显示当前航向和目标航向。
   · 醒目显示最短航向误差及其正负方向。
   · 显示当前舵角、指令舵角、偏航角速度和船速。
   · 小地图显示目标方向、实际船迹、船头方向和洋流漂移。

【九、状态面板与实时曲线，强制要求】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

页面底部或可展开侧栏提供半透明深色数据面板。桌面端不得遮挡主要驾驶区域，移动端允许折叠。

1. 主响应图：
   · 默认滚动显示最近 60 秒。
   · X 轴为时间，Y 轴标明单位。
   · 可通过图例开关目标航向、当前航向、误差、P/I/D 输出、指令舵角、实际舵角和偏航角速度。
   · 目标航向使用绿色虚线，当前航向使用蓝色，误差使用红色。
   · 支持暂停后悬停查看具体数值。

2. 数值状态：
   · 目标航向 ψ_ref（°）
   · 当前航向 ψ（°）
   · 偏差 e = wrap180(ψ_ref - ψ)（°）
   · 偏航角速度 r（°/s）
   · 指令舵角和实际舵角（°）
   · 积分累计误差 ∫e dt（°·s）
   · 原始与滤波后的 de/dt（°/s）
   · P、I、D 各项输出
   · 舵角是否饱和
   · 当前控制器类型和参数
   · 操控评分或实验指标

3. 动态教学提示示例：
   · “当前误差较大，P 项正在提供主要修正舵量。”
   · “误差持续同号，I 项仍在累积，观察紫色控制量逐渐增加。”
   · “当前误差已经很小，但 I 项仍保留非零舵量以抵消侧风。”
   · “误差正在快速减小，D 项开始反向制动，以降低超调。”
   · “舵角已饱和，但积分仍在增加，这就是积分饱和。”

【十一、技术规范与代码结构】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. 交付物必须只有一个 HTML 文件，内嵌 CSS 与 JavaScript。
2. 仅允许通过 CDN 加载库：
   · Three.js 稳定版本。
   · 可选 Tween.js。
   · 可选 Chart.js，或使用原生 Canvas 2D 绘图。
   · 如使用 Three.js 后期处理模块，也必须通过 CDN 加载。
3. 不得引用外部图片、HDR、纹理、3D 模型或音频文件。
4. 所有视觉资产必须由 Three.js 几何体、Shader、CanvasTexture、CSS、SVG 或程序化粒子生成。
5. 使用 requestAnimationFrame 渲染，固定 1/60 s 物理步长。
6. 将代码清晰划分并注释：
   · Configuration
   · Physics
   · PID Controller
   · Tutorial State Machine
   · Renderer
   · Ocean and Environment
   · Camera
   · Controls
   · Audio
   · Charts
   · UI
   · Main Loop
7. 控制器计算必须独立于渲染逻辑，便于切换 Manual / P / PI / PID。
8. 实验历史、指标计算和曲线数据应采用结构化对象管理。
9. 不得只制作界面占位符；所有模式、按钮、参数、课程阶段和图表必须实际工作。
10. 页面加载失败或 WebGL 不可用时提供明确错误提示。

【十二、默认实验与参数建议】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

为保证打开页面即可学习，请提供一组经过调试的默认参数和实验：

1. 默认教学目标：
   · 初始航向 0°。
   · 目标航向先从 0° 阶跃至 30°。
   · 随后在系统接近稳定时加入恒定侧风扰动。

2. P 实验：
   · 默认 Kp 应产生清楚可见但不过度振荡的响应。
   · 持续扰动后保留肉眼可见的稳态偏差。

3. PI 实验：
   · 使用与 P 相同的 Kp，再加入适中的 Ki。
   · 稳态误差最终接近零，但相较 P 出现更多超调或更长调节过程。

4. PID 实验：
   · 在 PI 基础上加入适中的 Kd 和微分滤波。
   · 相比 PI 明显减少超调和调节时间。

5. 参数数值需根据最终物理模型实际调试，不得机械照搬通用参数。

【十三、完成标准】
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

体验模式：
✓ 画面达到高质量网页 3D 小游戏水平，三种时段差异显著。
✓ 海面、船体、尾迹、天空、光照和环境具有丰富层次。
✓ 第一/第三视角切换流畅。
✓ 键鼠和舵轮控制手感清晰，满舵有延迟，回舵后有惯性。
✓ 风力和洋流变化能从船体运动、海面和小地图中共同观察到。
✓ 高画质下仍有合理性能，并可自适应降档。

教学模式：
✓ 学习者可依次完成手动、P、PI、PID 四阶段实验。
✓ P 阶段能清楚看到误差直接产生比例舵量，以及持续扰动下的稳态偏差。
✓ PI 阶段能看到积分累计、静差消除、超调和积分饱和。
✓ PID 阶段能看到 D 项提前抑制趋势、减少超调，同时展示噪声敏感性。
✓ P、I、D 三项输出可实时分解并与总舵角对应。
✓ 相同扰动可以重复运行，并能叠加比较 P、PI、PID 曲线。
✓ 自动计算上升时间、超调量、调节时间、稳态误差、IAE 和控制能量。
✓ 动态提示和阶段总结能把观察到的现象与控制理论准确对应。

工程质量：
✓ 单一 HTML 文件可直接运行。
✓ 不使用外部图片、模型或音频资源。
✓ 固定步长物理仿真稳定。
✓ 航向跨越 0°/360° 时误差和曲线处理正确。
✓ 桌面与移动端布局可用。
✓ 所有关键控件和课程流程均为真实功能，不是装饰性占位。
